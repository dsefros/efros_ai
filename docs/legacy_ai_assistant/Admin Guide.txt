1. Общее описание
Данное приложение предназначено для автоматизации анализа задач и интеграции с Redmine, YandexGPT и Telegram.
Код и все необходимые компоненты размещены локально в директории /opt/AI_assistent.
В качестве среды запуска используется Python 3.10+ в виртуальном окружении.

2. Запуск системы
Подключитесь к серверу под пользователем с правами на запуск приложения (например, root или sudoer).

Перейдите в корневую директорию проекта:

bash
Копировать
Редактировать
cd /opt/AI_assistent
Активируйте виртуальное окружение:

bash
Копировать
Редактировать
source venv/bin/activate
Запустите основное приложение:

bash
Копировать
Редактировать
python3 main.py
Для запуска в фоновом режиме рекомендуется использовать tmux, screen или nohup:

bash
Копировать
Редактировать
nohup python3 main.py > logs/app.log 2>&1 &
3. Остановка системы
Если приложение запущено в текущем терминале — остановите с помощью Ctrl+C.

Если приложение запущено в фоне, найдите PID и завершите процесс:

bash
Копировать
Редактировать
ps aux | grep main.py
kill <PID>
Для автоматизации можно настроить systemd-сервис (пример ниже).

4. Мониторинг и логирование
Логи записываются в файл:

bash
Копировать
Редактировать
/opt/AI_assistent/logs/app.log
В случае проблем изучите логи на наличие ошибок или предупреждений.

Для просмотра лога в реальном времени используйте:

bash
Копировать
Редактировать
tail -f /opt/AI_assistent/logs/app.log
5. Резервное копирование
Что нужно резервировать:
Директорию с проектом /opt/AI_assistent (код, конфиги, скрипты).

Файл .env с ключами и конфигурациями.

Логи по необходимости.

Рекомендуемый способ резервного копирования:
Создавайте архив:

bash
Копировать
Редактировать
tar czvf ai_assistent_backup_$(date +%F).tar.gz /opt/AI_assistent
Храните резервные копии в отдельном безопасном месте.

6. Обновление системы
Остановите текущее приложение (см. раздел 3).

Внесите изменения в код вручную или обновите файлы.

Обновите зависимости, если нужно:

bash
Копировать
Редактировать
source venv/bin/activate
pip install -r resources/requirements.txt --upgrade
Перезапустите приложение (см. раздел 2).

При изменении или добавлении документации .md индекс будет обновлен автоматически при следующем запуске.

7. Устранение неисправностей
Приложение не запускается или падает — проверьте логи.

Ошибки авторизации к Redmine или Telegram — проверьте корректность API ключей в .env.

Ошибки доступа к YandexGPT — проверьте токен и folder_id.

Проблемы с индексом документации — проверьте наличие .md файлов и права на запись в директории.
