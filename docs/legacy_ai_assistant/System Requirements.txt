Требования к оборудованию и ПО (System Requirements)
1. Минимальные требования к оборудованию
Компонент	Минимальные характеристики
Сервер / хостинг	CPU: 2 ядра (x86_64), RAM: 4 ГБ, Диск: 20 ГБ SSD
Клиентские рабочие станции	CPU: 1 ядро, RAM: 2 ГБ
Сеть	Стабильное интернет-соединение с пропускной способностью не менее 10 Мбит/с для API-вызовов и загрузки документов

2. Операционная система
Сервер: Linux (рекомендуется Ubuntu 20.04+ или аналогичный дистрибутив)
Клиенты: Windows 10/11 или Linux (для запуска клиентских утилит и взаимодействия через браузер/терминал)
Для локальной разработки допускается использование Windows с WSL 2

3. Программное обеспечение
Компонент	Версия / Требования
Python	3.10 (рекомендуется использовать виртуальное окружение)
Redmine	Установленная и доступная API версия Redmine 4.2+ (REST API)
Yandex Cloud SDK	yandex_cloud_ml_sdk, актуальная версия
Telegram Bot API	Telegram Bot, созданный в BotFather, с доступом к API
Дополнительные библиотеки Python	requests, redminelib, retry, python-dotenv, pathlib
Веб-сервер (опционально)	Nginx/Apache для развертывания (если применяется)

4. Сетевые требования
Доступ к API Redmine по HTTPS (порт 443)
Доступ к Telegram API по HTTPS (порт 443)
Доступ к Yandex Cloud API по HTTPS (порт 443)

Разрешённый исходящий трафик на вышеперечисленные сервисы
Внутренние настройки firewall/прокси должны позволять установку соединений с внешними API

5. Требования к хранению данных
Минимум 20 ГБ свободного пространства для кэша индексов, логов и документов
Резервное копирование каталогов с индексами и логами (рекомендуется)

6. Дополнительные рекомендации
Рекомендуется выделенный сервер для запуска бэкенд-сервисов проекта
Мониторинг состояния сервиса (CPU, RAM, сетевые ошибки)
Регулярное обновление библиотек и SDK для безопасности и стабильности
