1. Общее описание
Проект AI Assistant — это система автоматизации поддержки клиентов и сопровождения задач, интегрированная с Redmine для управления задачами, Telegram для уведомлений и Yandex GPT для интеллектуального анализа и генерации рекомендаций на основе корпоративной документации.

Основная цель — ускорить и улучшить качество обработки инцидентов и заявок, предоставляя технически точные рекомендации, основанные на документации и истории задач.

2. Компоненты системы и их взаимодействие
2.1. Клиент Redmine (clients/redmine/client.py)
Отвечает за получение задач из Redmine, фильтрацию по статусам и проектам.

Позволяет добавлять приватные комментарии в задачи.

Обеспечивает безопасную работу с API Redmine, включая повторные попытки и обработку ошибок.

2.2. Клиент Telegram (clients/telegram/client.py)
Отвечает за отправку уведомлений в Telegram-чат.

Форматирует сообщения с использованием HTML-разметки.

Имеет маппинг проектов на инженеров, добавляя их теги в сообщения.

Предоставляет методы для отправки уведомлений об успешной обработке задач и ошибках.

2.3. Клиент Yandex GPT (clients/yandex_gpt/client.py)
Отвечает за загрузку и кеширование поискового индекса на основе Markdown-документов.

Интегрируется с Yandex Cloud ML SDK для создания и работы с моделью Yandex GPT.

Формирует контекстный запрос для анализа задачи, передавая его в модель.

Получает и обрабатывает ответ, возвращая структурированный технический анализ и рекомендации.

3. Архитектура данных
Документация хранится в формате Markdown (resources/documents/markdown), а также в исходных PDF (resources/documents/pdfs).

Проекты и инженеры сопоставляются через табличный файл resources/documents/ingeners_projects.txt.

Кэш поискового индекса хранится в clients/yandex_gpt/index_id.txt и index_hash.txt.

4. Поток обработки задачи
Получение задач — RedmineClient извлекает задачи по заданным проектам и статусам.

Анализ задачи — YandexGPTClient формирует запрос на основе описания задачи и отправляет его в Yandex GPT для получения технического анализа.

Отправка уведомлений — TelegramNotifier рассылает в Telegram чат структурированное сообщение с анализом, добавляя тег инженера проекта.

Обновление задачи — при необходимости добавляет приватные комментарии в задачу Redmine.

5. Технические детали и зависимости
Язык: Python 3.10

Внешние библиотеки:

redminelib — для работы с Redmine API

requests — для HTTP запросов к Telegram API

yandex_cloud_ml_sdk — SDK для Yandex GPT

retry — для автоматических повторных попыток запросов

python-dotenv — для загрузки конфигурации из .env

Логирование настроено через стандартный logging с выводом в файл logs/app.log.

Виртуальное окружение — venv.

6. Безопасность
Все ключи и токены хранятся в .env и не коммитятся в репозиторий.

Запросы к API выполняются с TLS, в некоторых случаях с отключённой проверкой сертификата (для Redmine).

Взаимодействие с Telegram ограничено на уровне чатов по chat_id.
